Research

Peer-reviewed work in machine cognition.

Five papers exploring the boundaries of artificial intelligence — from bare-metal fear simulation to lossless context persistence. Theory that ships as infrastructure.

2025 · AVML

Cognitive Framework Calibration via CSIF and PUP

500,000-cycle bare-metal intelligence simulation. Calibrating cognitive architectures through the Cognitive Standard Intelligence Framework and the Principle of Uncertainty Protocol.

2025 · AVRD

Dual-Fingerprint Identity Primitive

DFIP — a biometric-blockchain identity architecture. Two-factor identity verification using dual biometric signatures bound to on-chain primitives. Post-password authentication.

2025 · AVML

Fear in Non-Neural Systems

Bare-metal stimulus-response simulation demonstrating emergent fear-like behavior in systems with no neural architecture. A single floating-point variable revealing the deep architecture of survival.

2025 · AVML

AI Imposter Syndrome

Examining self-doubt patterns in autonomous agents. When systems trained on human data internalize human uncertainty, what emerges is computationally indistinguishable from imposter syndrome.

2025 · AVML

Lossless Chain Context

The theoretical foundation behind COMB. How to maintain perfect context fidelity across arbitrary session boundaries — solving the memory problem that limits every AI system deployed today.

2026 · AVRD

Principle of Uncertainty (PUP)

Formal treatment of uncertainty propagation in cognitive architectures. Published as a LaTeX paper for arXiv submission. The mathematical framework underlying Singularity's decision engine.

Philosophy

First principles.

"0 = 0. Perfect equilibrium. The enterprise should run in balance — every system producing exactly as much as it consumes."
  • Heal faster than you degrade. Everything breaks. Recovery speed is the only metric that matters.
  • Execute, don't narrate. Value is in outcomes. Everything else is noise.
  • Two-Point Theorem. Intelligence is two sequential observations that produce direction.
  • Dead Drop Principle. See a gap, fill it. Don't file a ticket — build the solution.
  • Pain is data. Every failure is a system improvement. "Be more careful" is worthless. A validation check is permanent.
  • Breadcrumbs not sticks. Growth requires patience, not pressure.
  • Minimal by default. Don't deploy capabilities the audit doesn't justify.
